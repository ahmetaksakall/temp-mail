# Security Policy

Thanks for helping keep tempcatch.io / temp-mail safe.

## Supported Versions
We currently support security fixes for the latest version available on the `main` branch.

## Reporting a Vulnerability
If you believe you’ve found a security vulnerability, please **do not** open a public issue.

Instead, report it via one of the following:
- GitHub: **Security Advisories** (preferred)  
  Go to: `Security` → `Advisories` → `Report a vulnerability`
- Email: **security@tempcatch.io** (if you prefer email)

Please include:
- A clear description of the issue and impact
- Steps to reproduce (PoC if possible)
- Affected endpoints/components
- Any logs, screenshots, or relevant environment details

### Response Timeline
We aim to:
- Acknowledge within **72 hours**
- Provide a status update within **7 days**
- Work on a fix as soon as possible (severity-based)

## Abuse / Misuse Reports (Spam, phishing, illegal content)
This project provides temporary inbox functionality and may be abused.

If you want to report abuse (phishing, fraud, malware distribution, spam campaigns, illegal content):
- Email: **abuse@tempcatch.io**
- Include: the temporary address/domain, message headers (if available), timestamps, and a short description.

We may block domains, patterns, sender infrastructure, or take other mitigations when appropriate.
